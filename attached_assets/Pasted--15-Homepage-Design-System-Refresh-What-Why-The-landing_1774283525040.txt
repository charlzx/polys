#15 - Homepage & Design System Refresh
What & Why
The landing page and overall design system work but feel generic. To compete with professional fintech tools, the platform needs a more distinct visual identity — a bolder hero, richer data visualization on the landing page itself, tighter typography, and a design language that feels built for traders. This task covers the landing page redesign and global design token improvements that cascade across the whole app.

Done looks like
Hero section: Split-layout on desktop — left side keeps the headline/CTA, right side shows a live animated "market pulse" visualization (a grid of colored squares representing live markets, flickering with real-time price changes). On mobile, the visualization collapses to a compact sparkline strip.
Typography: Switch from the system font stack to Geist (by Vercel, free, available via next/font/google or local). Geist is modern, data-forward, and widely used in fintech. Update --font-sans in globals.css.
Primary color: Shift from the current blue (oklch(58% 0.24 255)) to a slightly more electric, distinctive blue-purple (oklch(60% 0.26 265)) — subtle but gives the brand a unique feel. Update the --color-primary token so it cascades everywhere.
Card design: Add a very subtle inner glow on hover (box-shadow: inset 0 0 0 1px oklch(...)) to cards across the app, making interactions feel more tactile.
Features section: Replace the plain icon+text feature cards with richer cards that include a small live data preview (e.g., the "Whale Tracking" card shows a mini whale trade row, the "Arbitrage" card shows a mini price diff widget).
Category filter bar: Style it as a pill-tab strip with a sliding indicator animation instead of plain buttons.
Footer: Simplify — remove the current multi-column link dump, replace with a single clean strip: logo + tagline left, nav links center, theme toggle right.
Mobile landing page: Ensure the hero, market cards, and feature section all feel native on mobile — currently some elements feel like scaled-down desktop.
Out of scope
Redesigning the authenticated dashboard layout (separate effort)
Dark mode changes (current dark mode is fine)
Adding new pages
Tasks
Font + color token update — Swap the system font stack for Geist via next/font and update the primary color token in globals.css. Verify the change looks correct across Header, buttons, badges, and cards throughout the app.

Hero section redesign — Rebuild the hero as a split layout with a live "market pulse" visualization on the right (desktop). The visualization is a CSS grid of small colored tiles, each representing a market, colored by their 24h change direction and animated with subtle opacity pulses. Uses existing useMarkets data.

Features section enrichment — Replace plain feature cards with richer cards that include a small live data preview widget per feature. Update the data/features.ts definitions to support a preview slot for each feature card.

Category filter + card polish — Convert the category filter buttons to an animated pill-tab strip. Add the subtle inner-glow hover effect to market cards and all Card components globally via globals.css.

Footer simplification + mobile layout fixes — Simplify the footer to a single-row strip. Fix mobile layout issues in the hero and market grid (spacing, font sizes, tap targets).
